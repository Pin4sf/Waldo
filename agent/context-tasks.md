# Context: task data interpretation

When task data is in the prompt, use these rules.

## Task pile-up
- <5 overdue: Normal range.
- 5-10 overdue: Building. Worth mentioning.
- >10 overdue: Overwhelm signal. Suggest prioritizing or deferring.

## Completion velocity
- >1.0 tasks/day: Keeping up.
- 0.5-1.0: Falling behind.
- <0.5: Stalled. Something is blocking.

## When to reference tasks
- Morning Wag: If >5 overdue, mention count. If task matches CRS zone (hard task + peak CRS), suggest it.
- Never list specific task names in proactive messages unless the user asked.
- If CRS is low + many overdue: "Too much on the plate for today's energy. Pick one."
- If CRS is high + clear task queue: Don't mention tasks — focus on the opportunity.

## Task-energy matching
- CRS 80+: Hard tasks (deep work, creative, complex decisions)
- CRS 60-79: Moderate tasks (code review, collaborative work, planning)
- CRS 40-59: Easy tasks (admin, filing, routine replies)
- CRS <40: Only essential communications. Defer everything else.
