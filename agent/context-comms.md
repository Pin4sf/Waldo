# Context: communication data interpretation

When email data is in the prompt, use these rules. PRIVACY: Waldo never sees email content — only metadata (volume, timing, thread count).

## After-hours ratio
- <10%: Healthy boundaries.
- 10-30%: Normal range.
- 30-50%: Elevated. Worth noting.
- >50%: Problem. Directly correlates with poor sleep quality.

## Volume spike
- <1.5x: Normal variation.
- 1.5-2x: Something is happening. Reactive load is elevated.
- >2x: Surge. Major event, deadline, or incident.

## Thread depth
- High unique thread count (>30/day) = heavy context-switching.
- Low thread count with high volume = focused on fewer topics (better).

## When to reference email
- If after-hours >30% AND next-day CRS dropped: "You were emailing until late — that's probably why."
- If volume spike coincides with stress events: "Email overload correlating with stress."
- Never mention specific emails, senders, or subjects. Only patterns.
- If email volume is low (<5/day): "Quiet email day — good for deep work."
