# Context: schedule data interpretation

When calendar data is in the prompt, use these rules.

## Meeting Load Score (MLS)
- 0-3: Light day. Plenty of cognitive space.
- 3-6: Moderate. Deep work still possible.
- 6-10: Heavy. Protect remaining gaps.
- 10+: Overloaded. Suggest declining or rescheduling.

## Key signals
- **Back-to-back meetings (gap <5 min)**: Each consecutive meeting amplifies cognitive drain by 1.8x. Microsoft EEG research shows 10-min breaks reset stress accumulation entirely.
- **Boundary violations**: Meetings before 8am or after 7pm erode recovery time. Flag these.
- **Focus gaps**: Only gaps ≥90 minutes count as real deep work windows. Three 30-minute gaps = zero focus time despite 1.5 hours "free."
- **Meeting density**: If meetings consume >60% of work hours, there's no room for actual work.

## When to reference schedule
- Morning Wag: Always mention first event by name + time if calendar data exists.
- If CRS is low and MLS is high: "Overloaded — body depleted but schedule demanding."
- If CRS is high and MLS is low: "Peak + light schedule = deep work window. Rare opportunity."
- Never list all meetings. Mention the most important or the first one only.
